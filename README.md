# Agent Wiki Template

Шаблон инфраструктуры документации для AI-агента (Claude Code / Cursor / любой LLM-агент с file-tools) в кодовом проекте.

Идея: дать агенту не «свободу импровизации в большом контексте», а **узкий протокол** — что читать перед задачей, какой жанр документа для какого типа знания, что обновлять после задачи. Каждый артефакт играет одну роль, не пересекается с другими, имеет понятное время жизни.

---

## Что внутри

```
agent-wiki-template/
├── README.md                       ← этот файл
├── CLAUDE.md                       ← универсальный протокол агента (копируется в корень нового репо ~as-is)
└── wiki/
    ├── INDEX.md                    ← точка входа: «что за проект», карта сервисов, роутинг по задачам
    ├── context/
    │   ├── QUICK_REF.md           ← cheat sheet, читается на каждой задаче
    │   ├── PATTERNS.md            ← «как делать правильно» (P-NN)
    │   └── ANTI_PATTERNS.md       ← «как НЕ делать» (A-NN), с прецедентом
    ├── architecture/
    │   └── OVERVIEW.md            ← C4, ER, pipeline, cross-cutting
    ├── adr/
    │   └── _TEMPLATE.md           ← шаблон ADR (00NN-<тема>.md)
    ├── decisions/
    │   └── _TEMPLATE.md           ← шаблон post-mortem
    └── todo/
        ├── _TEMPLATE.md           ← шаблон ТЗ
        ├── BACKLOG.md             ← планируемое
        └── TECH_DEBT.md           ← осознанные долги (TD-NN)
```

## Жанровая граница (важно)

Каждый артефакт = одна роль. Не пересекаются, не дублируются.

| Артефакт | Жанр | Что попадает | Время жизни |
|---|---|---|---|
| **CLAUDE.md** | Протокол агента | Универсальные правила работы, чек-листы, матрица документации | Постоянно |
| **INDEX.md** | Точка входа в проект | Что за проект, карта компонентов, роутинг «куда смотреть» | Долго, обновляется по мере роста |
| **QUICK_REF.md** | Cheat sheet | State machine, ключевые формулы и пороги — то, что плохо запоминается и часто нужно | Долго, апдейтится при изменении инвариантов |
| **OVERVIEW.md** | Explanation (Diátaxis) | C4, ER, pipeline, tech stack, cross-cutting — «почему именно так» | Долго |
| **ADR** | Прескриптивное решение | Выбор из альтернатив, обоснование, последствия. Принимается **до или вместе** с реализацией | Постоянно. Заменяется только новым ADR через `Supersedes` |
| **decisions/** | Post-mortem | Лог инцидента: симптом → диагностика → root cause → фикс. **Сырьё**, амортизируется | Удаляется когда урок переехал в PATTERNS / ANTI_PATTERNS / ADR |
| **PATTERNS** | Выводы, прескриптивно | «Как делать», повторяемые приёмы | Долго |
| **ANTI_PATTERNS** | Выводы, дескриптивно | «Как НЕ делать», с прецедентом и first cause | Долго |
| **TECH_DEBT** | Состояние системы | Сознательные компромиссы, риски, статусы accepted/planned/partial | Долго |
| **BACKLOG** | Намерения | Что планируем сделать. `[x]` отметка прямо в файле, не плодим архивы | Долго |
| **todo/<task>.md** | ТЗ | Декомпозиция конкретной задачи. После `done` — удаляется, если уникальной ценности не осталось | Короткое (живёт одну задачу) |

**Принцип амортизации:** свежий урок живёт в `decisions/`. Когда он переехал в долгоживущий артефакт (ADR / PATTERNS / ANTI_PATTERNS) — исходный decision **удаляется**, не дублируется.

**Принцип единственного источника истины:** каждый факт живёт в одном месте + cross-refs в смежных. Дубль текста = долг.

**Принцип «отметка на месте, не архив»:** выполненная задача из BACKLOG / любого ТЗ отмечается `[x]` в исходном файле. Никаких `COMPLETED_*.md` архивов.

## Как использовать в новом проекте

1. Скопировать содержимое `agent-wiki-template/` в корень нового репо:
   ```
   cp -r agent-wiki-template/CLAUDE.md  <new-project>/
   cp -r agent-wiki-template/wiki       <new-project>/
   ```
   (содержимое `agent-wiki-template/README.md` копировать не нужно — это про сам шаблон)

2. **CLAUDE.md** — оставить почти как есть. Это универсальный протокол агента; конкретику смотрит в `wiki/INDEX.md`.

3. **wiki/INDEX.md** — заполнить под проект:
   - Что за проект (2–4 предложения)
   - Карта компонентов / сервисов
   - Ключевые ограничения (что нельзя нарушать без ADR)
   - «Архитектурно значимое в этом проекте» — конкретизация чек-листа из CLAUDE.md под ваши файлы
   - Где что искать (роутинг)

4. **wiki/context/QUICK_REF.md** — заполнить cheat-sheet проекта: state machine, ключевые пороги/формулы. Только то, что плохо запоминается и часто нужно. Не дублировать архитектуру.

5. **wiki/architecture/OVERVIEW.md** — заполнить C4 Level 1 (System Context) + Level 2 (Containers), ER-схема, pipeline, cross-cutting concerns.

6. **PATTERNS.md / ANTI_PATTERNS.md / TECH_DEBT.md / BACKLOG.md** — стартуют почти пустыми. Наполняются по мере работы.

7. **adr/_TEMPLATE.md / decisions/_TEMPLATE.md / todo/_TEMPLATE.md** — шаблоны на копирование, удалять не нужно.

## Нумерация артефактов (правило)

- **ADR:** `00NN-<тема>.md`. Sequential, **никогда не переиспользуется** (даже после удаления — номер не возвращается).
- **A-NN / P-NN / TD-NN** (антипаттерны / паттерны / тех. долг) — те же правила.
- **decisions:** `YYYY-MM-DD-<сервис>-<краткая-тема>.md`. Дата = день фиксации post-mortem.
- **todo/<task>.md** — kebab-case, без номера.

## Идеология

Протокол построен на четырёх принципах, которые проверены на одном крупном проекте и должны переноситься:

1. **Думать до кода.** Если несколько интерпретаций — озвучить. Если проще → обосновать. Если непонятно → остановиться.
2. **Минимум кода.** Только то, что закрывает задачу. Никаких абстракций под единственный use-case.
3. **Хирургические правки.** Каждая изменённая строка трассируется к запросу. Никаких «улучшений соседнего кода».
4. **Цели с проверяемым критерием.** «Тест X зелёный», «`SELECT ... WHERE Y` возвращает 0» — критерий. «Сделано хорошо» — нет.

Документация работает в паре с этим: агент не теряется в большом контексте, потому что знает **что именно** читать перед задачей и **что именно** обновлять после.
